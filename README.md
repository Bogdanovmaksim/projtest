# Менеджер задач

Консольное приложение на C++20 для ведения списка задач. Проект позволяет хранить задачи в файле,
фильтровать большой список, сортировать задачи по дедлайну и переносить выполненные задачи в архив.

## Возможности

- добавление задач с описанием, дедлайном, категорией и уровнем важности;
- просмотр всех активных задач;
- фильтрация по ключевому слову, важности, статусу и ближайшим дням;
- сортировка задач по приближению дедлайна;
- редактирование любых данных существующей задачи;
- отметка задачи как выполненной;
- удаление задач;
- архивация завершенных задач в отдельный файл;
- сохранение и загрузка данных из файлов.

## Сборка и запуск

```bash
cmake -S . -B build
cmake --build build
./build/task_manager_app
```

При запуске программа попросит указать:

- файл задач;
- файл архива.

Если оставить поля пустыми и нажать Enter, будут использованы значения по умолчанию:

```text
tasks.txt
archive.txt
```

## Меню программы

```text
1. Добавить задачу
2. Показать все задачи
3. Фильтровать задачи
4. Показать задачи по дедлайну
5. Отметить выполненной
6. Удалить задачу
7. Изменить задачу
8. Архивировать выполненные
0. Сохранить и выйти
```

## Формат ввода

Дата вводится в формате:

```text
ДД.ММ.ГГГГ
```

Пример:

```text
15.06.2026
```

Уровень важности можно вводить по-русски:

```text
низкая
средняя
высокая
```

Статус при редактировании:

```text
активна
выполнена
```

## Формат данных

Задачи сохраняются в UTF-8 текстовом файле. Каждая задача хранится в отдельной строке:

```text
id|description|ДД.ММ.ГГГГ|category|importance|status
```

Служебные символы в описании и категории экранируются автоматически.

## Тесты

Тесты написаны с использованием Doctest и запускаются через CTest:

```bash
ctest --test-dir build --output-on-failure
```

Покрытие разделено по частям проекта:

- `tests/test_task_core.cpp` — даты, важность, статус, сериализация, фильтрация, сортировка;
- `tests/test_task_manager.cpp` — добавление, изменение, удаление, сохранение, загрузка, архив.

## Документация

Документация оформлена в стиле Doxygen. Основные комментарии находятся в заголовочных файлах:

- `include/task.h`;
- `include/task_manager.h`.

HTML-документацию можно собрать командой:

```bash
doxygen Doxyfile
```

## Распределение работы

- Участник 1: `include/task.h`, `src/task.cpp`, `tests/test_task_core.cpp`.
- Участник 2: `include/task_manager.h`, `src/task_manager.cpp`, `src/main.cpp`,
  `tests/test_task_manager.cpp`.

Такое разделение помогает показать, что у каждого участника есть собственный функциональный код,
тесты и документация.
